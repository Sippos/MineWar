# Project Vision

## One-sentence pitch

A fantasy mining-defense game where Warcraft III custom map energy meets digging, base upgrades, hero/faction identity, and fast playtest-driven iteration.

## Inspirations

- Hero Line Wars / Warcraft III fun-map flow: build momentum, defend, send pressure, scale income, and survive chaos.
- Mining/digging games: satisfying tunnels, resources, risk/reward, and returning loot to base.
- Small-session indie games: understandable in seconds, deep enough to replay.

## Core pillars

1. **Digging is the main verb.** The player should always have a meaningful reason to mine: resources, paths, shortcuts, defense, or enemy manipulation.
2. **The base is the strategic heart.** The base should be readable, upgradeable, and faction-flavored.
3. **Heroes/factions change plans.** Dwarf, Shaman, and future heroes should not just be skins; each should create different upgrade and defense choices.
4. **VS mode should feel like a fun map.** Sending enemies, scaling income, and timing pressure should be central.
5. **UI must be editable.** Core menus should be built as Godot scene nodes, not hidden in generated code.
6. **Small releases beat huge rewrites.** Every improvement should preserve the GitHub -> itch.io deploy loop.

## Near-term product target

The next milestone is a clean, stable playtest loop:

- start game,
- choose hero,
- mine and collect resources,
- open base menu,
- buy visible upgrades,
- survive enemy waves,
- return to menu or replay,
- deploy to itch.io without manual repair.

## Long-term ambition

Ship a polished, replayable first commercial indie game and build toward the first 10k players/revenue milestone through fast playtests, readable systems, and disciplined iteration.
